# 自建代理池：打造属于你的无限节点

想拥有一个永不过期、取之不尽的节点池吗？手把手教你如何通过几步简单的配置，构建自己的代理池，让你在任何时候都能轻松获取优质节点。

**正所谓授人以鱼不如授人以渔！**

---

## 🛠️ 准备工作

在开始之前，请确保你已具备以下条件：

1. 一台能运行 **Docker** 的设备（要求直连互联网，测试节点必须如此）。
2. 一个 **Github 账户** 或 **Cloudflare 账户**（并且已经完成域名托管，不要使用需要双向解析的免费域名）。

---

## 📦 涉及项目

* **Docker [subs-check](https://github.com/beck-8/subs-check)**
  该容器化工具会自动测速筛选节点。

---

## 🐳 1. 部署 Docker [subs-check](https://github.com/beck-8/subs-check)

**subs-check 支持三种部署方式，用户可根据自己的操作习惯和硬件环境选择最适合自己的方式：**

1. [小白命令一键部署容器](https://blog.cmliussss.com/p/ProxyPool/#1-1-%E9%83%A8%E7%BD%B2%E6%96%B9%E5%BC%8F%E4%B8%80%EF%BC%9A%E5%B0%8F%E7%99%BD%E5%91%BD%E4%BB%A4%E4%B8%80%E9%94%AE%E9%83%A8%E7%BD%B2%E5%AE%B9%E5%99%A8)
   适合新手，操作简单，但重置 Docker 后会**丢失所有配置信息**。
2. [使用 `docker-compose.yml` 部署容器](https://blog.cmliussss.com/p/ProxyPool/#1-2-%E9%83%A8%E7%BD%B2%E6%96%B9%E5%BC%8F%E4%BA%8C%EF%BC%9A%E4%BD%BF%E7%94%A8-docker-compose-yml-%E9%83%A8%E7%BD%B2%E5%AE%B9%E5%99%A8)
   推荐方式，**便于管理和持久化配置**，适合大多数用户。
3. [命令行创建 `docker` 容器](https://blog.cmliussss.com/p/ProxyPool/#1-3-%E9%83%A8%E7%BD%B2%E6%96%B9%E5%BC%8F%E4%B8%89%EF%BC%9A%E5%91%BD%E4%BB%A4%E8%A1%8C%E5%88%9B%E5%BB%BA-docker-%E5%AE%B9%E5%99%A8)
   适合熟悉 Docker 命令的用户，**subs-check 配置和节点信息可保留**，但**无法保留 Docker 构建信息**。

---

### 1.1 部署方式一：小白命令一键部署容器


| `docker run -d --name subs-check -p 8199:8199 -p 8299:8299 -e API_KEY=填入管理密码 --restart always ghcr.io/beck-8/subs-check:latest`<br/> |
| ------------------------------------------------------------------------------------------------------------------------------------------ |

例如：


| `docker run -d --name subs-check -p 8199:8199 -p 8299:8299 -e API_KEY=admin --restart always ghcr.io/beck-8/subs-check:latest`<br/> |
| ----------------------------------------------------------------------------------------------------------------------------------- |

---

### 1.2 部署方式二：使用 `docker-compose.yml` 部署容器

创建 `docker-compose.yml`，并启动容器。将 `config.yaml` 配置文件所在目录映射到容器 `/app/config` 内：


| `version: "3"`<br/>`services:`<br/>`  subs-check:`<br/>`    image: ghcr.io/beck-8/subs-check:latest`<br/>`    container_name: subs-check`<br/>`    # mem_limit: 500m`<br/>`    volumes:`<br/>`      - ${config 绝对路径}:/app/config`<br/>`      - ${output 绝对路径}:/app/output`<br/>`    ports:`<br/>`      - "8199:8199"`<br/>`      - "8299:8299"`<br/>`    environment:`<br/>`      - TZ=Asia/Shanghai`<br/>`      - API_KEY=admin`<br/>`      # - HTTP_PROXY=http://192.168.1.1:7890`<br/>`      # - HTTPS_PROXY=http://192.168.1.1:7890`<br/>`    restart: always`<br/>`    tty: true`<br/>`    network_mode: bridge`<br/> |
| ---------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------- |

---

### 1.3 部署方式三：命令行创建 `docker` 容器


| `docker run -d --name subs-check -p 8199:8199 -p 8299:8299 -v ${config 绝对路径}:/app/config -v ${output 绝对路径}:/app/output -e API_KEY=admin --restart always ghcr.io/beck-8/subs-check:latest`<br/> |
| ------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------- |

---

### 登录 **WebUI** 管理页面，编辑 `config.yaml` 配置文件

* WebUI：


  | `http://你设备的IP:8199/admin`<br/> |
  | ----------------------------------- |

你可以通过 **WebUI** 编辑你的`config.yaml`，参考文档：[config.yaml](https://github.com/beck-8/subs-check/blob/master/config/config.example.yaml)

> 你可以使用 [SubsCheck-Win-GUI](https://github.com/cmliu/SubsCheck-Win-GUI) 测试完善后将 `config.yaml` 然后替换原文件并重启容器即可完成配置。

* 通用订阅：


  | `https://你设备的IP:8299/download/sub`<br/> |
  | ------------------------------------------- |
* clash订阅：


  | `https://你设备的IP:8299/api/file/mihomo`<br/> |
  | ---------------------------------------------- |

## 🔐 2. 通过 **Gist** 实现**外网静态订阅**

* 获取 Github Token 与 Gist ID

### 2.1 如何生成 Github Token

1. 点击 Github 右上角头像，依次进入：`Settings` → `Developer settings` → `Personal access tokens` → `Tokens (classic)` → `Generate new token (classic)`；
   [![Settings](https://blog.cmliussss.com/img/ProxyPool/1-1.png)](https://blog.cmliussss.com/img/ProxyPool/1-1.png)
   [![Developer settings](https://blog.cmliussss.com/img/ProxyPool/1-2.png)](https://blog.cmliussss.com/img/ProxyPool/1-2.png)
   [![Generate new token (classic)](https://blog.cmliussss.com/img/ProxyPool/1-3.png)](https://blog.cmliussss.com/img/ProxyPool/1-3.png)
2. 在生成页面：将 **Expiration** 设置为 `No expiration`，勾选 `gist` 权限，然后点击 **Generate token**；
   [![Expiration](https://blog.cmliussss.com/img/ProxyPool/1-4.png)](https://blog.cmliussss.com/img/ProxyPool/1-4.png)
   [![gist](https://blog.cmliussss.com/img/ProxyPool/1-5.png)](https://blog.cmliussss.com/img/ProxyPool/1-5.png)
3. 生成后请务必记录这个 **Github Token**（仅显示一次）。
   [![github-token](https://blog.cmliussss.com/img/ProxyPool/1-6.png)](https://blog.cmliussss.com/img/ProxyPool/1-6.png)

---

### 2.2 如何获取 Gist ID

1. 登录 [gist.github.com](https://gist.github.com/)，创建一个新的 Gist；
   [![Gist](data:image/gif;base64,R0lGODlhAQABAIAAAAAAAP///yH5BAEAAAAALAAAAAABAAEAAAIBRAA7)](https://blog.cmliussss.com/img/ProxyPool/2-1.png)
2. 在生成的 Gist 页面，其链接格式类似：


| `https://gist.github.com/cmliussss2024/055c0f79d37f79235cda63c77a985749`<br/> |
| ----------------------------------------------------------------------------- |

* 其中 `055c0f79d37f79235cda63c77a985749` 就是你的 **Gist ID**。
  [![gist-id](https://blog.cmliussss.com/img/ProxyPool/2-2.png)](https://blog.cmliussss.com/img/ProxyPool/2-2.png)

---

## 📶 3. 通过 **CF隧道** 实现**外网动态订阅**

### 3.1 创建 CF 隧道，并获取隧道TOKEN

1. 登录 [Cloudflare](https://dash.cloudflare.com/)，进入 **Zero Trust**；
   [![Zero Trust](https://blog.cmliussss.com/img/ProxyPool/3-1-1.png)](https://blog.cmliussss.com/img/ProxyPool/3-1-1.png)
2. 点击 **网络** → **Tunnels** → **添加隧道**；
   [![添加隧道](data:image/gif;base64,R0lGODlhAQABAIAAAAAAAP///yH5BAEAAAAALAAAAAABAAEAAAIBRAA7)](https://blog.cmliussss.com/img/ProxyPool/3-1-2.png)
3. 选择隧道类型：`Cloudflared`；
   [![Cloudflared](data:image/gif;base64,R0lGODlhAQABAIAAAAAAAP///yH5BAEAAAAALAAAAAABAAEAAAIBRAA7)](https://blog.cmliussss.com/img/ProxyPool/3-1-3.png)
4. 为隧道命名，填入你运行**docker设备的名称**（如：`nas`），点击 **保存隧道**；
   [![保存隧道](https://blog.cmliussss.com/img/ProxyPool/3-1-4.png)](https://blog.cmliussss.com/img/ProxyPool/3-1-4.png)
5. 点击`运行命令`复制命令，如下：

  | ------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------- |

  [![复制命令](https://blog.cmliussss.com/img/ProxyPool/3-1-5.png)](https://blog.cmliussss.com/img/ProxyPool/3-1-5.png)
* 将命令末尾的TOKEN复制保存到本地，如下：


  | `eyJhIjoiOWIwM2EzMGM0NzFhOTUyMTk3YjhiOGJiNmYxMTNiZjYiLCJ0IjoiM2VmZjE3ZTYtNTVkNi00ZmEwLWEyZGUtNGE4NTZhNGFlMDE4IiwicyI6Ik0yRm1ZVEF6WlRJdE4yVXhZUzAwWTJJMUxXRmtOekV0TXpnMk5UQTROREl3T0RabSJ9`<br/> |

### 3.2 部署 CF 隧道

* 创建 `docker-compose.yml` ，并启动容器
  使用下面的配置来启动 Docker 容器，同时将 `隧道TOKEN` 替换为你复制保存的 **TOKEN**：


| `version: '3'`<br/>`services:`<br/>`  cloudflared:`<br/>`    container_name: Cloudflare_Tunnels`<br/>`    image: cloudflare/cloudflared:latest`<br/>`    restart: always`<br/>`    command: tunnel --no-autoupdate run --token [隧道TOKEN]`<br/> |
| ------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------ |

如：


| `version: '3'`<br/>`services:`<br/>`  cloudflared:`<br/>`    container_name: Cloudflare_Tunnels`<br/>`    image: cloudflare/cloudflared:latest`<br/>`    restart: always`<br/>`    command: tunnel --no-autoupdate run --token eyJhIjoiOWIwM2EzMGM0NzFhOTUyMTk3YjhiOGJiNmYxMTNiZjYiLCJ0IjoiM2VmZjE3ZTYtNTVkNi00ZmEwLWEyZGUtNGE4NTZhNGFlMDE4IiwicyI6Ik0yRm1ZVEF6WlRJdE4yVXhZUzAwWTJJMUxXRmtOekV0TXpnMk5UQTROREl3T0RabSJ9`<br/> |
| ----------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------- |

### 3.3 设置 CF 隧道的4个公共主机名

1. 路径**download**，用于通用订阅

* 子域：`任意值`（但4个主机名必须相同）
* 域：`任意值`（但4个主机名必须相同）
* 路径：`download`
* 类型：`HTTP`
* URL：`[你docker设备的内网IP]:8299`（注意端口为**8299**）
  [![3-3-3](https://blog.cmliussss.com/img/ProxyPool/3-3-3.png)](https://blog.cmliussss.com/img/ProxyPool/3-3-3.png)

---

2. 路径**api/file**，用于clash订阅

* 子域：`任意值`（但4个主机名必须相同）
* 域：`任意值`（但4个主机名必须相同）
* 路径：`api/file`
* 类型：`HTTP`
* URL：`[你docker设备的内网IP]:8299`（注意端口为**8299**）
  [![3-3-4](https://blog.cmliussss.com/img/ProxyPool/3-3-4.png)](https://blog.cmliussss.com/img/ProxyPool/3-3-4.png)

---

3. 路径**admin**，用于WebUI访问

* 子域：`任意值`（但4个主机名必须相同）
* 域：`任意值`（但4个主机名必须相同）
* 路径：`admin`
* 类型：`HTTP`
* URL：`[你docker设备的内网IP]:8199`（注意端口为**8199**）
  [![3-3-1](https://blog.cmliussss.com/img/ProxyPool/3-3-1.png)](https://blog.cmliussss.com/img/ProxyPool/3-3-1.png)

---

4. 路径**api**，用于WebUI获取信息

* 子域：`任意值`（但4个主机名必须相同）
* 域：`任意值`（但4个主机名必须相同）
* 路径：`api`
* 类型：`HTTP`
* URL：`[你docker设备的内网IP]:8199`（注意端口为**8199**）
  [![3-3-2](https://blog.cmliussss.com/img/ProxyPool/3-3-2.png)](https://blog.cmliussss.com/img/ProxyPool/3-3-2.png)
