## 一、使用前准备

在开始之前，你需要准备三样东西：

1. 一个对应系统的客户端；
2. 一个可用的订阅链接、配置文件或手动节点信息；
3. 一个能正常访问下载页面的网络环境。

常见导入方式有三种：

* 订阅链接导入；
* 本地配置文件导入；
* 二维码或手动添加节点。

如果你手里的订阅格式和客户端不兼容，就需要使用订阅转换工具。原项目也把订阅转换工具单独列为一类，用来解决 Clash、Mihomo、Sing-box、Surge、Quantumult X、Loon 等格式之间不兼容的问题。([GitHub](https://github.com/maomao533/dingyue-tools/tree/main "GitHub - maomao533/dingyue-tools: 精心整理的科学上网客户端工具合集，涵盖Windows、macOS、Linux、Android、iOS等全平台。所有工具均指向官方GitHub仓库的Releases页面，确保下载最新稳定版。 · GitHub"))

# 二、订阅转换工具教程

## 1. [subconverter](https://github.com/tindy2013/subconverter) 使用教程

[subconverter](https://github.com/tindy2013/subconverter) 是经典订阅转换工具，适合把一种订阅格式转换成另一种客户端可识别的格式。它的官方说明中写明支持 Clash、Quantumult X、Loon、SS、SSR、Surge、V2Ray 等多种格式作为源或目标格式，并提供了接口形式的转换方法。([GitHub](https://github.com/tindy2013/subconverter "GitHub - tindy2013/subconverter: Utility to convert between various subscription format · GitHub"))

### 适合谁用

适合需要多格式转换、多订阅合并、自建转换服务的人。不太适合完全不想接触参数的新手。

### 基础使用流程

第一步，打开 [subconverter 项目页面](https://github.com/tindy2013/subconverter)。

第二步，根据自己的系统下载 Releases 中对应版本。如果你只是普通用户，更推荐使用已经部署好的前端或服务端；如果你有服务器，可以自行部署。

第三步，准备你的原始订阅链接。

第四步，选择目标格式。例如：

* Clash / Mihomo 客户端：选择 `clash`
* Quantumult X：选择 `quanx`
* Loon：选择 `loon`
* Surge：选择 `surge`
* V2Ray：选择 `v2ray`

第五步，生成转换后的订阅链接，再复制到目标客户端中导入。

### 常见参数理解

subconverter 常见转换接口形式类似：

```text
/sub?target=目标格式&url=原订阅链接
```

实际使用时，原订阅链接通常需要进行 URL Encode，否则可能因为链接里有特殊符号导致转换失败。

### 常见问题

如果转换后无法导入，优先检查三点：

第一，原订阅链接是否有效。

第二，目标格式是否选错。

第三，订阅链接是否正确编码。

多个订阅合并时，不建议一次性放太多来源，先用一个订阅测试成功，再逐步增加。

## 2. [Sub-Store](https://github.com/sub-store-org/Sub-Store) 使用教程

[Sub-Store](https://github.com/sub-store-org/Sub-Store) 是高级订阅管理器，适合 Quantumult X、Loon、Surge、Stash、Shadowrocket 等用户。原项目介绍中也提到，Sub-Store 支持多订阅聚合、规则处理、格式转换和定时更新等功能。([GitHub](https://github.com/maomao533/dingyue-tools/tree/main "GitHub - maomao533/dingyue-tools: 精心整理的科学上网客户端工具合集，涵盖Windows、macOS、Linux、Android、iOS等全平台。所有工具均指向官方GitHub仓库的Releases页面，确保下载最新稳定版。 · GitHub"))

### 适合谁用

适合 iOS、macOS 深度用户，尤其是需要统一管理多个订阅、多个客户端配置的人。

### 基础使用流程

第一步，打开 [Sub-Store 项目页面](https://github.com/sub-store-org/Sub-Store)。

第二步，按项目说明部署或使用已有面板。

第三步，进入 Sub-Store 管理界面。

第四步，新建订阅，把原始订阅链接粘贴进去。

第五步，选择输出类型，例如 Clash、Surge、Loon、Quantumult X、Stash 等。

第六步，保存后复制生成的新订阅链接。

第七步，把新订阅链接导入对应客户端。

### 使用建议

Sub-Store 不只是转换工具，更适合长期管理订阅。你可以用它做节点筛选、重命名、分组、规则处理和定时更新。

新手第一次使用时，建议只做一件事：把一个原始订阅转换成一个目标客户端订阅。等确认能用后，再研究高级脚本和规则。

### 常见问题

如果生成的链接导入失败，先不要改复杂脚本。先测试最基础的转换流程：

原订阅链接 → Sub-Store → 生成目标订阅 → 客户端导入。

如果基础流程成功，再逐步添加规则处理。

## 3. [Hiddify Next](https://github.com/hiddify/hiddify-app/releases) 使用教程

[Hiddify Next](https://github.com/hiddify/hiddify-app/releases) 是跨平台客户端，支持 Windows、macOS、Linux、Android、iOS。原项目介绍中提到，它基于 Sing-box 内核，支持多平台、多协议、TUN 模式、远程配置文件和内置测速等功能。([GitHub](https://github.com/maomao533/dingyue-tools/tree/main "GitHub - maomao533/dingyue-tools: 精心整理的科学上网客户端工具合集，涵盖Windows、macOS、Linux、Android、iOS等全平台。所有工具均指向官方GitHub仓库的Releases页面，确保下载最新稳定版。 · GitHub"))

### 适合谁用

适合新手、多设备用户，以及想要一个客户端解决大部分配置问题的人。

### Windows / macOS / Linux 安装

第一步，打开 [Hiddify Next Releases](https://github.com/hiddify/hiddify-app/releases)。

第二步，下载与你系统对应的安装包：

* Windows 通常选择 `.exe` 或 `.msi`
* macOS 通常选择 `.dmg`
* Linux 可选择 `.AppImage`、`.deb` 或其他对应格式

第三步，安装并打开软件。

第四步，首次打开时，按系统提示允许网络权限或 VPN 权限。

### Android 安装

第一步，打开 [Hiddify Next Releases](https://github.com/hiddify/hiddify-app/releases)。

第二步，下载 Android APK。

第三步，安装 APK。如果系统提示“禁止安装未知来源应用”，需要在系统设置里允许当前浏览器或文件管理器安装应用。

第四步，打开 Hiddify Next，允许创建 VPN 配置。

### iOS 安装

iOS 用户可以通过 App Store 或项目提供的方式安装。若 App Store 地区无法搜索到，需要检查 Apple ID 地区。

### 导入订阅

打开 Hiddify Next 后，一般可以通过以下方式导入：

* 粘贴订阅链接；
* 扫描二维码；
* 导入配置文件；
* 从剪贴板识别配置。

导入成功后，先进行一次测速，再选择延迟较低的节点连接。

### 使用建议

新手建议先使用默认模式，不要一开始就改 DNS、TUN、高级路由等选项。如果能正常使用，再逐步调整规则和分流设置。

### 常见问题

如果连接后没有效果，检查：

* 是否允许 VPN 权限；
* 是否开启系统代理或 VPN 模式；
* 节点是否测速正常；
* 订阅是否已经过期；
* 是否被其他代理软件占用了端口。

## 4. [sing-box-converter](https://github.com/dzhuang/sing-box-converter) 使用教程

[sing-box-converter](https://github.com/dzhuang/sing-box-converter) 是偏向 Sing-box 配置转换的工具，适合把其他格式转成 Sing-box 可用配置。

### 适合谁用

适合主要使用 Sing-box、Hiddify Next、Karing 或其他 Sing-box 前端的用户。

### 基础使用流程

第一步，打开 [sing-box-converter 项目页面](https://github.com/dzhuang/sing-box-converter)。

第二步，查看项目 README 的使用方式。

第三步，准备原始配置或订阅链接。

第四步，按照工具要求输入源配置。

第五步，生成 Sing-box JSON 配置。

第六步，把生成的配置导入 Sing-box 或支持 Sing-box 的客户端。

### 使用建议

如果你只是新手，不建议直接手写复杂 JSON。可以先用 Hiddify Next 或 Karing 这类图形客户端导入订阅，等熟悉后再尝试单独转换配置。

### 常见问题

Sing-box 配置对 JSON 格式要求较严格。如果导入失败，常见原因是：

* JSON 格式错误；
* 缺少必要字段；
* 协议字段不兼容；
* DNS 或路由配置写错。

# 八、通用排错方法

无论你使用哪个工具，遇到问题时都可以按这个顺序排查。原项目的快速开始指南也建议按“选择客户端、下载安装、导入订阅、配置客户端、测试连接”的流程操作，并在连接异常时检查配置、节点、本地网络和防火墙等因素。([GitHub](https://github.com/maomao533/dingyue-tools/tree/main "GitHub - maomao533/dingyue-tools: 精心整理的科学上网客户端工具合集，涵盖Windows、macOS、Linux、Android、iOS等全平台。所有工具均指向官方GitHub仓库的Releases页面，确保下载最新稳定版。 · GitHub"))

## 1. 订阅导入失败

先检查：

* 订阅链接是否复制完整；
* 链接是否过期；
* 客户端是否支持该格式；
* 是否需要先转换格式；
* 本地网络是否能访问订阅地址。

解决方式：

* Clash 类客户端使用 Clash/Mihomo 格式；
* Sing-box 类客户端使用 Sing-box JSON；
* Surge、Loon、Quantumult X 使用各自兼容格式；
* 格式不对时使用 subconverter 或 Sub-Store 转换。

## 2. 连接成功但网页打不开

先检查：

* 是否开启系统代理或 VPN；
* 是否选中了可用节点；
* 是否有其他代理软件冲突；
* 浏览器是否安装了独立代理插件；
* DNS 是否异常；
* 当前模式是否选错。

建议先切换到 Global 测试。如果 Global 能用、Rule 不能用，说明大概率是规则分流问题。

## 3. 手机后台自动断开

Android 检查：

* 电池优化；
* 后台运行权限；
* VPN 常驻权限；
* 系统管家限制。

iOS 检查：

* 是否手动关闭了 VPN；
* 是否切换网络导致断开；
* App 是否需要重新授权 VPN 配置。

## 4. 节点测速全部失败

常见原因：

* 订阅已经失效；
* 网络环境不稳定；
* 客户端内核异常；
* 防火墙拦截；
* DNS 解析失败。

可以尝试：

* 更新订阅；
* 更换网络；
* 重启客户端；
* 换一个客户端测试；
* 查看日志定位错误。

实际使用中，不需要每个工具都安装。最合理的方式是：每个平台保留一个主力客户端，再准备一个订阅转换工具备用。这样既方便管理，也能减少系统代理冲突和配置混乱。
